# Google Shopping Review Crawler v29

## 목적
Google Shopping 미국 화면에서
- 브랜드명
- 상품명
- 리뷰 시작 날짜
- 리뷰 끝 날짜(비우면 오늘까지)
를 입력하면 해당 상품의 사용자 리뷰를 수집해서 CSV로 다운로드하는 로컬 웹앱입니다.

## 주요 기능
- 브랜드명 + 상품명 키인 검색
- 날짜 범위 필터링
- 리뷰 날짜 상대표현(`6 days ago`, `1 week ago`)을 실제 날짜로 변환
- `Most recent` 정렬 시도
- `More reviews` / `Show more` 계열 버튼 반복 클릭
- CSV 다운로드
- 작업 상태 / 로그 / 미리보기 제공
- 일시 중단 / 재개 / 중단 후 삭제

## 실행 방법
1. 처음 실행: `01_install_and_run_web.bat`
2. 이후 실행: `02_run_web.bat`
3. 브라우저에서 `http://127.0.0.1:8765`

## 입력 규칙
- 브랜드명: 예) `Hankook`
- 상품명: 예) `iON evo AS SUV`
- 리뷰 시작 날짜: `2025.12.01` / `2025-12-01` / `2025/12/01`
- 리뷰 끝 날짜: 비우면 오늘까지

## 출력 컬럼
1. `No.`
2. `브랜드명`
3. `상품명`
4. `리뷰 계정명`
5. `별점`
6. `날짜`
7. `리뷰 내용`
8. `출처 웹사이트`

## 주의
- Google UI 구조가 바뀌면 일부 클릭 동작이 깨질 수 있습니다.
- 이 버전은 DOM 클래스 고정 의존을 줄이기 위해 **텍스트 기반 클릭 + 일반화된 리뷰 추출** 방식으로 작성했습니다.
- Google에서 지역/동의창/봇체크가 뜰 수 있으므로, 처음에는 **브라우저 창 보면서 실행(headful)** 을 권장합니다.
- 검색 결과에서 완전히 동일 상품 카드가 여러 개 보이면, 리뷰 수가 동일한 카드 중 한 개만 선택해 리뷰 영역으로 진입하도록 설계했습니다.

## 권장 테스트 순서
1. headful 체크
2. `Hankook` / `iON evo AS SUV`
3. 시작 날짜만 입력
4. 결과가 정상인지 먼저 1회 검증
5. 그다음 다른 상품으로 확대


추가 파일
- `03_check_environment.bat`: Python/패키지/포트 상태 점검용
- `server_boot.log`: 서버 시작 실패 시 원인 로그
- `open_when_ready.py`: 서버가 실제로 준비된 뒤에만 브라우저를 여는 헬퍼
- `launch_server.py`: 서버 시작 전 포트 충돌/예외를 더 명확하게 보여주는 실행 파일


## 실행 우선순위
- 처음이든 나중이든 가장 먼저 `01_install_and_run_web.bat`를 눌러도 됩니다.
- `02_run_web.bat`도 이제 누락 패키지가 있으면 자동으로 설치를 다시 시도합니다.
- 에러가 나면 터미널 마지막 줄과 `server_boot.log`를 확인하세요.

## CAPTCHA / Google verification
- Google verification 화면이 뜨면 브라우저 창에서 직접 해결합니다.
- 해결 후 웹앱의 **인증 완료 후 Resume** 버튼을 누르면 크롤링이 다시 이어집니다.
- headless 모드에서는 수동 확인이 불가능하므로 verification이 뜨면 headful 모드로 다시 실행하세요.
